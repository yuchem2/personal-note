## 개요
프로그램을 **수학적 함수의 계산**으로 취급하고, 데이터의 상태를 바꾸는 것보다 **새로운 값을 만들어내는 것**에 집중하는 프로그래밍 패러다임.

- 순수 함수형 언어: Haskell, Lisp, OCaml
- 멀티 패러다임(함수형 지원): Javascript, Python, [Kotlin](Kotlin.md), Swift, Scala, Java(최신 버전)
## 핵심 개념
- **순수 함수**: 입력이 같으면 항상 같은 출력을 내놓고 다른 객체의 상태를 변경하지 않고 함수 외부나 바깥 환경과 상호작용하지 않는 함수
- **불변성**: 한 번 정해진 데이터는 절대 바뀌지 않습니다. 값을 수정하고 싶다면 원본을 고치는 대신, 변경사항이 반영된 새로운 복사본을 생성합니다.
- **일급 객체로서의 함수**: 함수를 변수에 담거나, 다른 함수의 인자로 전달하고, 함수 자체를 결과로 반환할 수 있습니다.
- **고차함수**: 함수를 인자로 받거나 함수를 반환하는 함수
- **선언적 패턴**: `for`나 `while` 루프를 사용해 일일이 명령하는 대신, 데이터가 어떻게 변환되어야 하는지 흐름을 기술합니다.
## 장점
- 상태 변화가 상태 변화가 없으므로 **코드 예측이 쉽고 버그가 적습니다.**
- 공유되는 상태가 없고 데이터가 불변하므로, 여러 프로세스가 동시에 같은 데이터에 접근해도 데이터가 꼬이는 *레이스 컨디션*이 발생하지 않습니다. 멀티코어 CPU 활용에 매우 유리합니다.
- *순수 함수*는 외부 환경에 의존하지 않으므로, 함수 그 자체만 떼어내서 **테스트하기가 매우 쉽습니다.** 입력값만 넣으면 결과가 정해져 있어 예측 가능한 코드가 됩니다.
- *고차 함수*(`map`, `filter`, `reduce`)를 활용하면 복잡한 로직을 마치 레고 블록을 조립하듯 짧고 명확하게 작성할 수 있습니다. 반복적인 `if-for` 문을 걷어내어 코드의 가독성이 높아집니다.
## 단점
- 모든 데이터를 복사해서 새로 만들기 때문에 **메모리 사용량이 늘어날 수 있음.**
- 객체 지향 방식에 익숙하다면 사고방식을 전환하는 데 시간이 걸립니다.
