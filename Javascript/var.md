ES6에서 [let](let.md), [const](const.md)가 도입되기 전까지 자바스크립트에서 유일한 변수를 선언하는 키워드

```js
var score; // score가 가리키는 주소에는 undefined가 할당된다.
```
위와 같은 식은 두 단계에 걸쳐 [변수 선언](변수%20선언.md)을 수행한다.
- **선언 단계**: 변수 이름을 등록해서 자바스크립트 엔진에 변수의 존재를 알린다.
- **초기화 단계**: 값을 저장하기 위한 메모리 공간을 확보하고 암묵적으로 `undefined`를 할당해 초기화한다.

```js
var score = 80;

// 위 코드와 아래 코드는 정확하게 동일하게 동작한다.
var score;
score = 80;
```
> 엄밀하게 말하면, 위 코드는 둘 다 할당이 아니라 [[변수 재할당]]이다. 선언문에서 `undefined`로 할당을 하기 때문이다.

변수 선언과 함께 변수 할당을 할 수 있고, 분리할 수도 있다. 하지만 위의 두 코드는 정확하게 동일하게 동작한다.
1. 먼저 `score` 변수 이름을 등록한다.
2. `score` 변수에 값을 저장하기 위한 메모리 공간을 확보하고 `undefined`를 할당한다.
3. `score` 변수에 `80`을 저장한다.

```js
console.log(score); // undefined
var score = 80;
console.log(score); // 80
```

또한, [호이스팅](호이스팅.md)에 의해 `score`는 해당 선언이 있는 어디서든 접근이 가능하지만, 실제 값이 `80`이 되기 위해서는 `var score = 80;` 구문이나 `score = 80;` 구문 이후에서 실제로 `score`의 값이 `80`이 된다.


## 특징
- 블록 레벨 스코프를 지원하지 않고, 함수 레벨 스코프를 지원한다.
