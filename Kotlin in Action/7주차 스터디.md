## 질문 리뷰
- `in` / `out` 위치가 헷갈려서 코드를 작성해보니 파라미터가 함수 타입인 경우 그 타입의 파라미터 자리에 T를 넣으면 `out`으로, 반환 값 자리에 T를 넣으면 `in`으로 해야 오류가 발생하지 않더라고요. 해당 경우에 대해서 설명해주실 수 있나요?
    - 알아보고 추후에 설명드리도록 하겠다.
- 공식 문서에 `out`, `in` 설정을 무시하라는 지시어 `@UnsafeVariance`라는 어노테이션이 있더라고요. 해당 어노테이션을 사용하는 것이 문제라고 생각이 드는데, 이러한 어노테이션을 사용하는 경우가 있나요? 
    - 러스트와 같은 경우도 비슷한 역할이 있는데, 허락하지 않은 코드도 수행될 수 있음
    - 개발자가 문제가 생기면 내가 책임질테니까 강제하는 문법
    - `!!`와 유사하다고 생각하면 됨.
- 런타임에서 타입이 지워지면서 발생하는 문제가 있는가?

    - 컬렉션 - 제네릭 관련을 많이 사용하다보면 문제가 생김. 
    - 대표적인 것은 컬렉션 사용할 때나 람다를 사용할 때..
