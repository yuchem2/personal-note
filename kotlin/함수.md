```kotlin
fun max(a: Int, b: Int): Int {
    return if (a > b) a else b
}
```
- 함수는 `fun` 키워드를 통해 선언할 수 있다. 
- 모든 코틀린 파일의 최상위 수준에 정의할 수 있다. (특별한 상태나 인스턴스 메서드가 없는 클래스의 생김을 방지하기 위해)
- 함수에 전달하는 인자 중 일부 또는 전부의 이름을 명시할 수 있다. 모든 인자의 이름을 지정하는 경우 순서를 변경할 수도 있다.
- [디퐅트 파라미터](디퐅트%20파라미터.md)를 지정할 수 있다.

> #### 최상위 함수가 Java에서도 문제없이 동작하는 이유
> JVM은 클래스 안에 들어아있는 코드만을 실행할 수 있기 때문에 코틀린 컴파일러는 소스 파일 이름(`join.kt` -> `JoinKt`)을 가진 클래스를 자동으로 생성한다. 코틀린 파일의 모든 최상위 함수는 이 클래스의 정적 메서드가 된다.
> 
> 만약 이러한 이름을 변경하고 싶다면 파일 수준의 `@file:JvmName(...)`을 파일의 맨 앞, 패키지 이름 선언 이전에 추가하면 된다. 
## `main` 함수

```kotlin
fun main(args: Array<String>) {
    println("Hello, world!")
}
```
- 최상위에 있는 `main` 함수를 애플리케이션의 진입점으로 지정할 수 있다.
- `main` 함수는 인자를 생략할 수 있다.
- `main` 함수는 아무 값도 반환하지 않는다.
## 함수 종류
![문과 식](문과%20식.md)
- 블록 본문 함수: 본문이 중괄호로 둘러싸인 함수
    - 값을 반환하는 경우 반드시 반환 타입을 지정하고, `return` 문을 사용해 반환값을 명시해야 한다.
- 식 본문 함수: 본문이 등호와 식으로 이루어진 함수 `fun max(a: Int, b: Int): Int = if (a > b) a else b`
    - 값을 반환하는 경우 반환 타입을 지정하지 않아도 [타입추론](Kotlin.md#타입%20추론)으로 타입이 지정된다.
## 디폴트 파리미터 함수
함수 파라미터에 디폴트 파라미터 값을 전달하여 인자의 기본값을 정의할 수 있다. 일반적으로 뒤에서부터 정의하는 것이 권장되지만, 기술적으로는 불가능하지 않다.

함수의 디폴트 파라미터 값은 함수를 호출하는 쪽이 아니라 함수 선언 쪽에 인코딩된다. 

> #### 기본값과 자바
> 자바에는 디폴트 파라미터 값이라는 개념이 없어 기본값을 제공하는 코틀린 함수를 자바에서 호출하는 경우에는 모든 인자를 명시해야 한다.
> 
> 만약 해당 함수를 자주 호출해야 해서 자바 측에서 편하게 사용하기 위해서는 `@JvmOverloads` 어노테이션을 함수에 추가할 수 있다. 해당 어노테이션은 코틀린 컴파일러가 자동으로 맨 마지막 파리미터로부터 파라미터를 하나씩 생략한 오버로딩한 자바 메서드를 추가해준다.
> 
> ```kotlin
> @JvmOverloads
> fun <T> joinToString(
>     collection: Collection<T>,
>     separator: String = ", ",
>     prefix: String = "",
>     postfix: String = ""
> ): String { /* ... */ }
> ```
> ```java
> String joinToString(Collection<T> collection, String separtor, String prefix, String postfix);
> String joinToString(Collection<T> collection, String separtor, String prefix);
> String joinToString(Collection<T> collection, String separtor);
> String joinToString(Collection<T> collection);
> ```
> 
> 각각의 오버로드 함수들은 시그니처에서 생략된 파라미터에 대해 코틀린 함수의 디폴트 파라미터 값을 사용한다.
## 가변 인자 함수
리스트 [Collections](Collections.md)을 생성하는 함수(`listOf<T>`)는 원하는 만큼 많이 원소를 전달할 수 있다. 

```kotlin
fun listOf<T>(vararg values: T): List<T> { /* 구현 */}
```

자바에서는 가변 인자를 표현할 때 타입 뒤에 `...`을 붙이지만 코틀린에서는 파라미터 앞에 `vararg` 변경자를 붙여 표시한다. 

```kotlin
fun main(args: Array<strings>) {
    val list = listOf("args: ", *args) // 스프레드 연산자: *
    println(list)
}
```

이미 배열에 들어있는 원소를 가변 길이 인자로 넘길 때 자바에서는 그냥 넘기면 되지만 코틀린에서는 배열을 명시적으로 풀어 배열의 각 원소가 인자로 전달되게 해야 한다. 
## 로컬 함수
코틀린에서는 함수 내부에 함수를 정의할 수 있다. 이러한 함수를 로컬 함수라고 한다.
```kotlin
class User(val id: Int, val name: String, val address: String)

fun saveUser(user: User) {
    fun validate(value: String, feildName: String) {
        if (value.isEmpty()) {
            throw IllegalArgumentException(
                "Can't save user ${user.id}: " +
                "empty $fieldName"
            )
        }
    }
    
    validate(user.name, "Name")
    validate(user.address, "Address")
    
    // user를 데이터베이스에 저장한다.
}
```

로컬 함수는 자신이 속한 바깥 함수의 모든 파라미터와 변수를 사용할 수 있다.
