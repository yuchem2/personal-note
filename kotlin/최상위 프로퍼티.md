[함수](함수.md)와 마찬가지로 [프로퍼티](프로퍼티.md)도 파일 최상위 수준에 놓을 수 있다. 이런 프로퍼티 값은 정적 필드에 저장된다.

```kotlin
var opCount = 0

fun performOperation() {
    opCoun++
    // ...
}

fun reportOperationCount() = println("Operartion performed $opCount times")
```

최상위 프로퍼티를 활용하여 코드에서 상수를 정의할 수 있다. 다른 모든 프로퍼티처럼 접근자 메서드를 통해 자바 코드에 노출된다. 

```kotlin
val UNIX_LINE_SEPERATOR = "\n"
```

만약 상수를 자연스럽게 `public static final` 필드로 노출하고 싶다면 `const` 변경자를 추가하면 된다. **(기본 타입과 `String` 타입의 프로퍼티만 `const`로 지정할 수 있다.)**

```kotlin
const val UNIX_LINE_SEPERATOR = "\n"
// 자바에서는 다음과 같다.
public static final String UNIX_LINE_SEPERATOR = "\n";
```

> 코틀린 표준 라이브러리도 몇 가지 유용한 최상위 함수와 프로퍼티를 제공한다. 
> 예를 들어 kotlin.math 패키지에서는 `max`, `E`, `PI`와 같은 함수와 상수를 제공한다.
