```kotlin
fun ignoreNulls(str: String?) {
    val strNotNull: String = str!!
    println(strNotNull.length)
}
```

> 코틀린에서는 해당 연산자를 의도적으로 못생기게 만들었고, 이는 이 연산자를 일반적인 상황에서는 사용하지 않는 것이 좋은 것을 의미한다. 해당 연산자만을 사용하려면 널이 아닌 타입으로 지정하는 것이 맞다.

코틀린에서 [널이 될 수 있는 타입](널이%20될%20수%20있는%20타입.md)의 값을 다룰 때 사용할 수 있는 연산자로, 어떤 값이든 널이 아닌 타입으로 강제로 바꿀 수 있다. 실제 `null`에 사용하면 NPE이 발생한다. 또한, `!!`를 `null`에 대해서 사용하면 스택 트레이스에서 어떤 파일의 몇 번째 줄인지는 확인할 수 있지만, 어느 식에서 발생했는지는 확인할 수 없다.
