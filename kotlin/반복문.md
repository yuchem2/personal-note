## while
[Kotlin](Kotlin.md)에는 `while`과 `do-while` 구문이 있다. 두 루프의 문법은 다른 언어에서와 유사하다.
```kotlin
while (조건) {
    /* ... */
    if (shouldExit) break
}

do {
    if (shouldSkip) continue
    /* ... */
} while (조건) // 처음에는 본문을 무조건 한번 실행한다.
```
- `break`은 자신을 둘러싼 루프를 중단시킨다.
- `continue`는 루프의 다음 단계를 시작한다.

내포된 루프의 경우 코틀린에서는 레이블을 지정할 수 있다. 그 후 `break`나 `continue`를 사용할 때 레이블을 참고할 수 있다. 레이블은 `@` 기호 다음에 식별자를 붙인 것이다.

```kotlin
outer@ while (outerCondition) {
    while (innerCondition) {
        if (shouldExitInnter) break
        if (shouldSkipInner) continue
        if (shouldExit) break@outer
        if (shouldSkip) continue@outer
    }
}
```
## for
[Kotlin](Kotlin.md)에서는 C와 같은 `for` 루프가 없고, Python과 같은 범위를 이용한다. 범위를 쓸 때는 `..` 연산자를 사용한다. 이때 범위는 폐구간이다.

```kotlin
val oneToTen = 1 .. 10
```

범위를 활용하면 다음과 같이 반복문을 정의할 수 있다.

```kotlin
for (i in 1 .. 100) { /* ... */ } // 1부터 100까지 반복
for (i in 100 downTo 1 step 2) { /* ... */ } // 100부터 -2씩 1까지 반복
for (i in 0 ..< size) { /* ... */ } // 0부터 size-1까지 반복
for (i in 0 .. size-1) { /* ... */ } // 0부터 size-1까지 반복
for (char in 'A' .. 'F') { /* ... */ } // 'A'부터 'F'까지 반복
```
- `1..100`는 정방향 범위를 만든다.
- `100 downTo 1`는 역방향 순열을 만든다.
- `step 2`은 증가 값의 절대 값을 주어진 값으로 변경한다.

객체에 대한 이터레이션도 다음과 같이 수행할 수 있다.

```kotlin
val collection = listOf("red", "green", "blue")
for (color in collection) { /* ... */ }
// 인덱스와 함께 컬렉션을 이터레이션 할 수 있다.
for ((index, element) in collection.withIndex()) { /* ... */ }

val binaryReps = mutableMapOf<Char, String>()
// 키를 letter에 값을 binary에 매핑할 수 있다.
for ((letter, binary) in binaryReps) { /* ... */ }
```

> #### `in` 키워드 활용
> `for`에서 `in` 키워드를 활용하면 해당 원소가 컬렉션에 있는 원소 중 하나이고, 주어진 범위에 따라 원소 순회를 하게 된다. 
> 
> 하지만, `for`가 아닌 다른 곳에서 사용되는 경우 해당 범위에 속하는 지 여부를 반환하는 역할로 변경된다.
> ```kotlin
> c in 'a' .. 'z' // a <= c && c <= z로 변환되어 boolean 값을 반환
> c !in 'a' .. 'z' // a > c && c > z 로 변환될 수 있다.
> // 범위에만 국한되지 않고 비교가 가능한 클래스라면 인스턴스 객체를 통해 범위를 만들수 있다.
> "Kotlin" in "Java" .. "Scala" // "Java" <= "Kotlin" && "Kotlin" <= "Scala"
> "Kotlin" in setOf("Java", "Scala") // 이 집합에는 "Kotlin"이 없다.
> ```
> 
> 비교가 가능한 클래스를 이용해 비교를 할 때에는 내부적으로 `Comparable` 인터페이스를 구현한 클래스여야 한다.
