## 개요
코틀린은 자바 가상머신(JVM) 플랫폼과 JVM 외의 다른 플랫폼에서 돌아가는 현대 프로그래밍 언어이다.

처음에는 "더 나은 자바"를 목표로 시작했다. 즉, 개발자들이 일반적으로 저지를 수 있는 유형의 오류를 방지하고, 현대적 언어 설계 패러다임을 포용하면서도 자바가 쓰이던 모든 곳에 더 편리하게 쓸 수 있는 언어를 목표로 하고 있다.
## 목적
코틀린의 주목적은 현재 자바가 사용되고 있는 모든 용도에 적합하면서도 더 간결하고 생산적이며 안전한 대체 언어를 제공하는 것이다. 
## 철학
- 코틀린은 실용적인 언어이다.
    - 실제 문제를 해결하기 위해 만들어진 실용적인 언어
    - 다른 프로그래밍 언어가 채택한 이미 성공적으로 검증된 해법과 기능에 주로 의존
    - 도구를 강조
- 코틀린은 간결하다.
    > #### 언어가 간결하다
    >  작성된 코드를 읽을 때 의도를 쉽게 파악할 수 있는 구문 구조를 제공하고, 그 의도를 달성하는 방법을 이해할 때 방해가 될 수 있는 부가적인 준비 코드가 적다.
    - 객체지향 언어에 존재하는 여러가지 번거로운 준비 코드(게터, 세터, 생성자 로직 등)을 암시적으로 제공
    - 람다와 익명 함수 제공
- 코틀린은 안전하다.
    - `Null PointException`를 방지
    - 클래스 캐스트 예외를 방지한다. (타입 검사와 캐스트를 한 연산자로 수행)
- 코틀린은 상호운용성이 좋다.
    - 코틀린의 클래스나 메서드를 일반적인 자바 클래스나 메서드와 똑같이 사용할 수 있다.
    - 거의 대부분을 자바 표준 라이브러리 클래스에 의존

## 맛보기
- 클래스 본문을 지정하지 않고도 프로퍼티가 포함된 데이터 클래스를 정의할 수 있다.
- `val` 키워드를 사용해 읽기 전용 프로퍼티를 선언한다.
- 인자의 기본값을 제공한다.
- 타입 시스템을 통해 널이 될 수 있는 값들을 명시적으로 다룰 수 있다.
- 클래스 안에 포함될 필요가 없는 최상위 함수를 정의할 수 있다.
- 함수나 생성자를 호출할 때 파라미터 이름을 지정한 인자를 사용한다.
- 람다식을 사용해 컬렉션 연산을 활용한다.
- 엘비스 연산자(`?:`)를 사용해 변수가 `null`일 때에 대비한 값을 제공한다.
- 문자열 템플릿을 제공한다.
- 데이터 클래스를 위해 `toString` 같은 함수를 사용할 수 있다.
## 주요 특성
코틀린은 다중 패러다임 언어이다. 코틀린은 [정적 타입 지정 언어](정적%20타입%20지정%20언어.md)이다. 또한, 객체지향 언어와 함수형 언어의 아이디어를 조합하였다.
### 정적 타입 지정 특징
#### 타입 추론
코틀린에서는 컴파일 시점에 프로그램에 사용된 모든 식의 타입이 알려져 있어야 하지만, 프로그래머가 직접 모든 변수의 타입을 명시할 필요는 없다. 대부분의 경우 **코틀린 컴파일러가 문맥으로부터 변수 타입을 자동으로 유추할 수 있기 때문이다.**

```kotlin
val x: Int = 1 // 변수 타입을 명시적으로 지정 가능
val y = 1      // 변하지만 명시적으로 지정하지 않아도 되는 경우
```
#### 널이 될 수 있는 타입
널이 될 수 있는 타입을 지원하여 *널 포인트 예외*가 발생하는지 여부를 컴파일 시점에 검사할 수 있어 프로그램의 신뢰성을 좀 더 높일 수 있다.
### [함수형 프로그래밍](함수형%20프로그래밍.md)과 객체지향의 조합
함수형을 택할 때는 대표적으로 "코드 중복 회피", "안전항 동시성", "테스트 하기 쉬운 코드"라는 장점이 있다. 코틀린은 쉽게 함수형 프로그래밍 기법을 적용하기 위해 다음과 같은 기능을 제공한다.
- 함수 타입을 지원함에 따라 함수를 일급 객체로 다룰 수 있다.
- 람다식을 지원함에 따라 쉽게 코드 블럭을 정의하고 여기저기 전달하기 쉽다.
- 멤버 참조를 통해 함수를 값으로 사용할 수 있다.
- 데이터 클래스는 불변적인 데이터를 저장하는 클래스를 만들 수 있는 간결한 구문을 제공한다.
- 코틀린 표준 라이브러리는 객체와 컬렉션을 함수형 스타일로 다룰 수 있는 API를 제공한다.

```kotlin
messages
    .filter { it.body.isNotBlank() && !it.isRead }
    .map(Message::sender)
    .distinct()
    .sortedBy(Sender::name)
```

### 코루틴(coroutine)
코틀린은 동시성과 비동기 프로그래밍의 문제를 코루틴이라 불리는 일시 중단 가능한 계산을 사용해 접근한다. 코루틴에서는 코드가 자신의 실행을 잠시 중단시킬 수 있고, 나중에 (중단했던 지점부터) 작업을 계속 수행할 수 있다. 또한, 코루틴은 아주 가벼운 추상화라 **큰 성능 손해를 보지 않고 수백만 개의 동시성 작업ㅇ르 시작할 수 있다.**

```kotlin
suspend fun processUser(credentials: Credentials) {
    val user = authenticate(credentials)
    val data = loadUserData(user)
    val profilePicture = loadImage(data.imageID)
    // ...
}

// suspend 키워드를 통해 각 함수가 호출되어도 블록되지 않도록 할 수 있다.
suspend fun authenticate(c: Credentials): User { /* ... */ }
suspend fun loadIUserData(u: User): Data { /* ... */ }
suspend fun loadImage(id: Int): Image { /* ... */ }
```

네트워크 호출 요청은 무한정 길어질 수 있고, 각각의 네트워크 요청을 수행할 때 `precessUser` 함수의 실행은 일시 중단되고 결과를 기다리게 된다. 하지만 이 코드가 실행되는 스레드는 블록되지 않는다. `processUser`의 결과를 기다리는 동안, 스레드는 사용자의 입력에 반응하는 등의 다른 작업을 수행할 수 있다.

블로킹 없이 위 코드를 한 번에 하나씩 순차적으로 실행하는 코드를 명령형 스타일로 작성할 수는 없다. 반면 콜백이나 반응형 확장을 사용하면 이런 단순한 연속적인 로직을 더 복잡한 코드로 바꿀 수 있다.

```kotlin
suspend fun loadAndOverlay(first: String, second: Striing): Image = 
    coroutineScope {
        val firstDeferred = async { loadIamge(first) }
        val secondDeferred = async { loadImage(second) }
        combineImage(firstDeferred.await(), secondDeferred.await())    
    }
```

위 코드는 두 프로세스(함수) 중 하나라도 실패하면 자동으로 다른 한쪽은 취소된다. 
## 주요 사용처
- 백엔드 지원: 코틀린 서버 프로그래밍
    - HTML 페이지를 전송하는 웹 애플리케이션
    - JSON API를 제공하는 백엔드 애플리케이션
    - RPC 프로토콜이나 메시지 버스를 통해 서로 통신하는 작은 서비스로 이루어진 마이크로서비스
    - 주요 기술: [Spring]([https://kotlinlang.org/docs/jvm-spring-boot-restful.html](https://kotlinlang.org/docs/jvm-get-started-spring-boot.html), [Ktor](https://ktor.io), [http4k](https://http4k.org/)
> #### Ktor는 코틀린의 DSL 기능을 사용하여 HTTP 요청을 라우팅하는 방법을 이용한다.
> 
> DSL(Domain Specific Language)은 코틀린 언어 특성을 유연하게 조합하여 설정, 복잡한 객체 생성, 객체 관계 매핑(ORM) 작업, 객체를 데이터베이스 포현으로 변환하거나 역방향으로 변환하기 등에 자주 사용된다.
- 모바일 개발: 안드로이드는 코틀린 우선
    - 안드로이드는 2017년부터 코틀린을 앱 빌딩을 위한 언어로 공식 지원
    - 주요 기술: [KTX 라이브러리](https://developer.android.com/kotlin/ktx), [젯팩 컴포즈 툴킷](https://developer.android.com/jetpack/compose)
    - 다중 플랫폼: iOS, JVM, JS 및 그 외의 플랫폼에서 사용 가능
## 컴파일
코틀린은 컴파일 언어로, 여러 타깃을 대상으로 컴파일할 수 있다.
- JVM에서 실행되는 JVM 바이트코드
- 추가 변환 후 안드로이드에서 실행되기 위한 JVM 바이트코드
- 다른 운영체제에서 네이티브로 실행되기 위한 네이티브 타깃
- 브라우저에서 실행되기 위한 자바스크립트 또는 웹 어셈블리
- ... etc
### 컴파일 과정
코틀린 컴파일러는 코틀린 소스 코드(`.kt`)를 분석하여 `.class` 파일을 만들어낸다. 만들어진 `.class` 파일은 특정 애플리케이션 유형에 맞는 패키징 과정을 거쳐 실행될 수 있다.

```bash
kotlinc <소스파일 또는 디렉터리> -include-runtime -d <jar 이름>
java -jar <jar 이름>
```

JVM은 원래 코드가 어떤 형식으로 컴파일되었는지에 무관하게 `.class` 파일을 실행할 수 있다. 그러나 코틀린 내장 클래스와 API는 의존관계로, 코틀린 런타임 라이브러리라는 추가 정보가 필요하다.
