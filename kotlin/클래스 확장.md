기존 자바 코드와 코틀린 코드를 자연스럽게 통합하는 코틀린의 핵심 목표 중 하나를 이루기 위해 기존 자바 프로젝트에 있는 코드를 처리할 수 있어야 한다. 기존 Java API를 재작성하지 않고 편한 기능을 사용할 수 있기 위해 **확장 함수, 프러퍼티** 기능이 제공된다.
## 확장 함수
개념적으로 확장 [함수](함수.md)는 어떤 클래스의 멤버 메서드인 것처럼 호출할 수 있지만 그 클래스의 밖에서 선언된 함수다.

```kotlin
fun String.lastChar(): Char = this.get(this.length - 1)
```

추가 함수 앞에 그 함수가 확장할 클래스의 이름만 덧붙이면 쉽게 구현할 수 있다. 
- 수신 객체 타입: 확장할 클래스 이름 
- 수신 객체: 확장 함수 호출 시 호출하는 대상 값(객체)
- 문법상 반드시 짧은 이름을 써야 한다.

> 내부적으로 확장 함수는 수신 객체를 첫 번째 인자로 받는 정적 메서드다. 따라서 확장 함수를 호출해도 다른 어댑터 객체나 실행 시점 부가 비용이 들지 않는다. 덕분에 자바에서도 확장 함수를 사용하기도 편하다.
> ```java
> char c = StringUtilKt.lastChar("java");
> ```

추가 함수를 호출하는 구문은 일반적인  클래스 멤버를 호출하는 구문과 동일하다.

```kotlin
fun main() {
    // 수신 객체 타입 = String
    // 수신 객체 = "Kotlin"
    println("Kotlin".lastChar())
}
```

확장 함수를 통해 어느 클래스에나 원하는 메서드를 추가할 수 있다. `final`로 상속이 불가능한 경우에도 문제가 없다. **자바 클래스로 컴파일된 클래스 파일이 있는 한 원하는 대로 확장을 추가할 수 있다.**

확장 함수 본문에서 `this`를 사용할 수 있고, 생략할 수도 있다. 또한, 일반적인 인스턴스 메서드의 내부와 마찬가지로 수신 객체의 메서드나 프로퍼티를 바로 사용할 수 있다.

하지만 **캡슐화**를 깨지는 않는다. 확장 함수 내부에서는 클래스 내부에서만 사용할 수 있는 **비공개(`private`) 멤버나 보호된(`protected`) 멤버를 사용할 수 없다.**

확장 함수를 다른 패키지에서 쓸려면 다른 클래스나 함수와 마찬가지로 해당 함수를 임포트해야만 한다. 

확장 함수는 단지 정적 메서드 호출에 대한 문법적인 편의이다. 확장 함수는 정적 메서드와 같은 특성을 가져 하위 클래스에서 오버라이드할 수 없다. 이름과 파라미터가 완전히 같은 확장 함수를 정의할 수는 있지만, 실제 호출될 함수는 컴파일 시점의 타입에 의해 결정되고, 실행 시간에 결정되지 않는다.

> 어떤 클래스를 확장한 함수와 그 클래스의 멤버 함수의 이름과 시그니처가 같다면 확장 함수가 아니라 멤버 함수가 호출된다. (멤버 함수의 우선순위가 더 높음)
## 확장 프로퍼티
확장 함수와 마찬가지로 확장 프로퍼티를 추가할 수 있다. 프러퍼티라고 불리지만 실제로는 아무 상태를 가질 수 없다. 그러므로 항상 커스텀 접근자를 정의해야 한다.

```kotlin
val String.lastChar: Char
    get() = get(length - 1)
```

확장 함수와 마찬가지로 단지 프로퍼티에 수신 객체 클래스가 추가됬을 뿐이다. 뒷받침하는 필드가 없어 기본 게터 구현을 제공할 수 없으므로 **최소한 게터는 꼭 정의해야 한다.**

```kotlin
var StringBuilder.lastChar: Char
    get() = get(length - 1)
    set(value: Char) {
        this.setCharAt(length - 1, value)
    }
```

자바에서 위와 같은 확장 프로퍼티를 사용하고 싶다면 항상 `StringUtilKt.getLastChar("java")`나 `StringUtilKt.setLastChar(sb, '!')`와 같이 게터나 세터를 명시적으로 호출해야 한다.
