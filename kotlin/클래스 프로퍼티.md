```java
public class Person {
    private final String name;
    
    public Person(String name) {
        this.name = name;
    }
    
    public String getName() {
        return name;
    }
}
```

위의 자바 코드 클래스를 코틀린 클래스 코드로 옮기면 아래와 같이 간단하게 표현할 수 있다.

```kotlin
class Person(val name: String)
```
- 코틀린의 기본 가시성은 `public`이다. 

자바에서는 멤버 필드의 가시성은 보통 `private`이다. 접근해야할 필요가 생기면 접근자 메서드를 제공한다. 접근자 메서드에는 게터가 있고, 세터가 있을 수 있다. **자바에서는 필드와 접근자를 한데 묶어 프로퍼티**라고 부르며 프로퍼티 개념을 활용하는 프레임워크가 많다.

[Kotlin](Kotlin.md)은 프로퍼티를 언어 기본 기능으로 제공하며 자바의 필드와 접근자 메서드를 완전히 대신한다. 
- `val`로 선언된 프로퍼티는 코틀린에서 비공개 필드와 게터를 만들어낸다.
- `var`로 선언된 프로퍼티는 코틀린에서 비공개 필드와 게터, 세터를 만들어낸다.

> **게터와 세터 이름 규칙**
> - 기본: 프로퍼티 이름에 게터의 경우 접두어로 `get`이 붙고, 세터의 경우 `set`이 붙는다.
> - 예외: `is`로 시작하는 프로퍼티는 게터는 `get`이 붙지 않고, 세터는 `is`를 `set`으로 바꾼 이름을 사용한다.
>   
>  ex) `name` -> `getName()`, `setName()` | `isStudent()` -> `isStudent()`, `setStudent()`

```kotlin
fun main() {
    val person = Person("Bob", true)
    // 프로퍼티 이름으로 사용해도 자동으로 게터 호출
    println(person.name)
    println(person.isStudent)

    person.isStudent = false // 프로퍼티 이름을 사용해도 자동으로 세터 호출
    println(person.isStudent)
}

```
- 게터와 세터를 명시적으로 호출하지 않고 프로퍼티를 직접 사용해도 코틀린이 필요한 메서드를 자동으로 호출해준다.

대부분의 프로퍼티에는 그 프로퍼티의 값을 저장하기 위한 필드인 뒷받침하는 필드가 있다. 하지만 원한다면 프로퍼티 값을 그때그때 계산할 수도 있다.
## 프로퍼티 커스텀 접근자
커스텀 접근자를 만드는 일반적인 경우는 어떤 프로퍼티가 같은 객체 안의 다른 프로퍼티에서 계산된 직접적인 결과인 경우다. 이렇게 프로퍼티에 접근할 때 계산을 할 수 있는 **온더고(on the go) 프로퍼티**라고 한다.

```kotlin
class Rectangle(val heaight: Int, val width: Int) {
    val isSquare: Boolean
        get() {
            return height === width
        }
        // get() = height === width
}
```

커스텀 게터를 정의하는 방식과 클래스 안에 파라미터가 없는 함수를 정의하는 방식은 구현이나 성능 차이가 없다. 일반적으로 **클래스의 특성을 기술하고 싶다면 프로퍼티**로, **클래스의 행동을 기술하고 싶다면 멤버 함수**로 하는 것이 가독성이 좋다.
