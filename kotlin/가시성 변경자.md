가시성 변경자는 코드 기반에 있는 선언에 대한 [클래스](클래스.md) 외부 접근을 제어한다. 코틀린에서는 이를 위해 `public`, `protected`, `private`, `internal`을 제공한다. **기본적으로는 모두 `public`이다.**

| 변경자               | 클래스 멤버            | 최상위 선언           |
| ----------------- | ----------------- | ---------------- |
| `public` (기본 가시성) | 모든 곳에서 접근 가능      | 모든 곳에서 접근 가능     |
| `internal`        | 같은 모듈 안에서만 접근 가능  | 같은 모듈 안에서만 접근 가능 |
| `protected`       | 하위 클래스 안에서만 접근 가능 | - (사용 불가능)       |
| `private`         | 같은 클래스 안에서만 접근 가능 | 같은 파일에서만 접근 가능   |

코틀린에서는 가시성이 높은 함수에서 가시성이 낮은 [함수](함수.md)나 [프로퍼티](프로퍼티.md)를 호출할 수 없게 막고 있다. (`public`에서 `private` 호출) 이는 **어떤 클래스의 기반 타입 목록에 들어있는 타입이나 제네릭 클래스의 타입 파라미터에 들어있는 타입의 가시성은 그 클래스 자신의 가시성과 같거나 더 높아야 하고, 메서드의 시그니처에 사용된 모든 타입의 가시성은 그 메서드의 가시성과 같거나 더 높아야 한다**라는 일반적인 규칙에 해당한다.

> **코틀린의 가시성 변경자와 자바**
> 코틀린의 `public`, `private`, `protected`는 컴파일된 자바 바이트 코드 안에서 그대로 유지된다. 자바에서 똑같은 가시성을 사용한 경우와 같다. 유일한 예외는 `private` 클래스로, 자바에서는 `private` 클래스를 만들 수 없음으로 내부적으로 코틀린의 `private` 클래스를 패키지 전용 클래스로 컴파일한다.
> 
> `internal`은 패키지 전용 가기성과 전혀 다르고, 모듈은 보통 여러 패키지로 이뤄지며 서로 다른 모듈에 같은 패키지에 속한 선언이 들어가 있을 수 있다. 그러므로 바이트 코드 상에서는 `public`이 된다.
> 
> `protected` 또한, 자바에서는 같은 패키지 안에서 접근이 가능하기 때문에 코틀린과 다르게 동작한다.
> 
> 위와 같은 차이 때문에 코틀린에서는 접근 불가능한 경우가 자바에서는 접근할 수 있는 경우가 생긴다. 다만, 코틀린 컴파일러는 `internal`에 한해서 `internal` 멤버의 이름을 보기 나쁘게 바꿔 접근이 가능하지만 보기 안좋게 변경시킴으로서 호출을 방지한다.
