[Kotlin](Kotlin.md)은 클래스를 조직화하기 위해 패키지라는 개념을 사용한다. 모든 코틀린 파일 맨 앞에는  `package` 문이 올 수 있다. `package` 문이 있는 파일에 들어있는 모든 선언은 해당 패키지 안에 들어간다.

```kotlin
package geometry.shapes

class Rectangle(val heaight: Int, val width: Int) {
    val isSquare: Boolean
        get() = height === width
}

fun createUnitSquare(): Rectangle {
    return Rectangle(1, 1)
}
```

같은 패키지에 속해 있다면 다른 파일에서 정의한 선언일지라도 직접 사용할 수 있다. 반면 다른 패키지에 정의한 선언을 이용하려면 해당 선언을 `import` 문으로 불러와야 한다.

```kotlin
package geometry.example

import geometry.shapes.Rectangle
import geometry.shapes.createUnitSquare

fun main() {
    println(Rectangle(3, 4).isSquare)
    println(createUintSquare().isSquare)
}
```

패키지 이름 뒤에 `.*`을 추가하면 패키지 안에 모든 선언을 임포트할 수 있다. 이런 방식을 *스타 임포트*라고 한다. *스타 임포트*를 사용하면 패키지 안에 있는 모든 것을 불러오게 된다.
## 디렉토리 구조
자바에서는 패키지의 구조와 일치하는 디렉터리 계층 구조를 만들고 클래스의 소스코드를 그 클래스가 속한 패키지와 같은 디렉토리에 위치시켜야 한다.

```text
geometry
 ├ example
 │  └ main.java
 └ shaeps
    ├ Rectangle.java
    └ RectangleUtil.java
```

코틀린에서는 여러 클래스를 같은 파일에 넣을 수 있고, 파일의 이름도 마을대로 정할 수 있다.

```text
geometry
 ├ example.kt
 └ shaeps.kt
```

하지만 대부분의 경우 자바와 같이 패키지별로 디렉터리를 구성하는 편이 낫다. 특히 자바와 코틀린을 함께 사용하는 프로젝트에서는 자바의 방식을 따르는 것이 중요하다. 자바의 방식을 따르지 않으면 자바 클래스를 코틀린 클래스로 마이그레이션할 때 문제가 생길 수도 있다.
