```kotlin
// 람다식 문법
{ x: Int, y: Int -> x + y }

// 람다의 다양한 사용방법
{ println(42) }() // 람다식 직접 호출
people.maxByOrNull({ p: Person -> p.age }) // 정식 문법
people.maxByOrNull({ it.age }) // 축약
people.maxByOrNull { it. age } // 코틀린스러운 호출
val sum = { x: Int, y: Int -> 
    prinln("Computing the sum of $x and $y...")
    x + y
}
```

람다는 람다식이라고도 하며 기본적으로 다른 함수에 넘길 수 있는 작은 코드 조각을 의미한다. 람다식의 특징은 다음과 같다.
- 코틀린에서 람다식 문법은 항상 중괄호로 둘러싸여 있다. 
- 화살표를 통해 인자 목록과 본문을 구분한다. 
- 컴파일러는 람다 파라미터의 타입도 타입 추론할 수 있다. 
- 만약 람다식의 인자가 하나이고, 타입 추론 가능하면 `it`이라는 이름을 사용할 수 있다.
- 람다식은 직접 호출도 가능하지만, 이 경우 보통 [`run`](수신%20객체%20함수.md#run)을 사용하는 것이 더 일반적이다.
- [함수](함수.md) 호출 시 맨 뒤에 있는 인자가 람다식이라면 그 람다를 괄호 밖으로 빼낼 수 있다. 만약 람다가 어떤 함수의 유일한 인자라면 빈 괄호를 제거할 수 있다.
- 만약 둘 이상의 람다를 인자로 받는 함수는 둘 이상의 람다를 괄호 밖으로 빼낼 수 없다.
- 본문이 여러 줄로 이뤄진 경우 본문의 맨 마지막에 있는 식이 람다 값의 결과가 된다. [값을 반환하는 블록](값을%20반환하는%20블록.md)과 동일하다.
- 람다에는 [익명 객체](object%20키워드.md#객체%20식)와 달리 인스턴스 자신을 가리키는 `this`가 없다. 람다 안에서 `this`는 그 람다를 둘러싼 클래스의 인스턴스를 가리킨다.
## 변수 캡처
```kotlin
fun main() {
    var counter = 0
    val inc = { counter++ }
}

// 위 코드는 실제로 아래 코드처럼 동작한다.
class Ref<T>(var value: T)

fun main() {
    val counter = Ref(0)
    val inc = { counter.value++ }
}
```

람다를 함수 안에서 정의하면 함수의 파라미터 뿐만 아니라 람다 정의보다 앞에 선언된 로컬 변수까지 람다에서 모두 사용할 수 있다. 또한, **캡처한 변수(람다  안에서 접근할 수 있는 외부 변수)가 있는 람다를 저장해서 함수가 끝난 뒤에 실행해도 람다의 본문 코드는 캡처한 변수를 읽거나 쓸 수 있다.** 
- `final` 변수(`val`): 람다 코드를 변수 값과 함께 저장
- `final`이 아닌 변수(`var`): 변수를 `Ref` 클래스 인스턴스에 넣어 Ref 인스턴스 참조를 `final`로 만들어 람다 코드와 함께 저장한다.

```kotlin
// 이 함수는 항상 0을 리턴한다.
// onClick 핸들러는 나중에 실행되므로, return clicks 시점엔 아직 clicks가 증가하지 않았다.
fun tryToCountButtonClicks(button: Button): Int {
    var clicks = 0
    button.onClick { clicks++ }
    return clicks
}
```

**람다를 이벤트 핸들러나 다른 비동기적으로 실행되는 코드로 활용하는 경우 로컬 변수 변경은 람다가 실행될 때만 발생한다.** 그러므로, 만약 로컬 변수 변경을 추적하기 위해서는 다음과 같은 방법을 사용할 수 있다.
- 변경되는 변수를 함수 밖으로 옮기기
- `AtomicInteger`와 같은 객체로 래핑
- 콜백으로 결과를 전달
- `StateFlow` / `LiveData` 활용 (Android / Coroutine 환경)
- 변경되는 변수를 클래스 프로퍼티로 관리
