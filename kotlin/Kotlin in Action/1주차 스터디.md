## 질문 리뷰
- 객체지향 vs 함수형 패러다임 지표?
    - 70-80 퍼센트는 맞는 말 -> java로 만든 레거시 시스템을 kotlin으로 만드는 경우가 많음
    - 하지만 지금은 처음부터 kotlin으로 작성된 경우가 많음
    - 대부분 객체지향 패러다임을 지향하지만, 함수형만 지향하는 경우가 있음
    - Java에서 함수를 일급 객체로 사용하지 못하니까 JVM 기반 함수형으로 만들기 가장 좋은 책. 
    - 함수형 관련된 책이 많아서 생각보다 많고, 니즈도 많음
    - 지금은 소수이지만 함수형이 점차 늘지 않을까. 
- 코루틴 비동기에서 JS의 이벤트 루프 역할을 수행하는 객체가 있는가?
    - 내부적으로 이벤트 루프가 있음. But, 자세한 내용은 14장 이상에 나와있음. 지금은 어려우니 있다고 알고 있으면 좋음 
    - 코루틴의 dispatcher와 java의 virtual thread는 유사
        - Platform thread는 기존 java의 thread와 비슷.
            - 1개의 platform thread위에 n개의 VT위에 존재할 수 있음
            - 1개의 PT는 1개의 OS와 일치
        - context switch 비용을 거의 줄이기 위해 virtual thread를 도입한 것.
        - OS의 도움을 받지 않고, JVM 내에서 관리를 위임하면서 VT는 기존 OS Thread보다 비용이 적음
    - 위 컨셉과 유사하게 코루틴이 이벤트 루프를 수행함. (코루틴 디스패처가 어떻게 실행할 지 결정하는 것!)
    - 추가 학습을 위한 추천 도서는 [코틀린 코루린](https://product.kyobobook.co.kr/detail/S000210537188), 강의는 [코틀린 코루틴 완전 정복](https://www.inflearn.com/course/%EC%BD%94%ED%8B%80%EB%A6%B0-%EC%BD%94%EB%A3%A8%ED%8B%B4-%EC%99%84%EC%A0%84-%EC%A0%95%EB%B3%B5?cid=334944&srsltid=AfmBOoqKxtiasB6lorhDp5N_RzsfLr0yLAjs_dYGrFjAURKni1jR5c6r)
- 코틀린이 다른 언어로 빌드될 때 표준이 변경될 때마다 대응될 때 어떻게 해야하나?
