클래스 안에 널이 아닌 [프로퍼티](프로퍼티.md)를 생성자 안에서 초기화하지 않고 특별한 메서드 안에서 초기화 할 수 없다. 코틀린에서는 일반적으로 모든 [프로퍼티](프로퍼티.md)를 [생성자](생성자.md)에서 초기화해야 한다. 만약 생성자에서 초기화하지 않고, 다른 메서드에서 초기화 로직을 생성해야하는 경우 해당 [프로퍼티](프로퍼티.md)를 [널이 될 수 있는 타입](널이%20될%20수%20있는%20타입.md)으로 선언한 뒤 `null`로 초기화하거나 지연 초기화 프로퍼티를 사용할 수 있다.

```kotlin
class MyService {
    fun performAction(): String = "Action Done!"
}

@TestInstance(TestInstance.Lifecycle.PER_CLASS)
class MyTest {
    private lateinit var mytService: MyService
    
    @BeforeAll fun setUp() {
        myService = MyService()
    }
    
    @Test fun testAction() {
        assertEquals("Action Done!", myService.performAction())
    }
}
```

> `lateinit` 프로퍼티가 반드시 클래스의 멤버일 필요는 없다. 함수 본문 안의 지역 변수나 최상위 프로퍼티도 지연 초기화할 수 있다.

`lateinit` 변경자를 통해 프로퍼티를 나중에 초기화할 수 있다. 이때 `lateinit` 변경자를 붙이면 항상 `var`로 지정해야 한다. 이러한 프로퍼티를 초기화 전에 사용하면 `Kotlin.UninitializedPropertyAccessException: lateinit property * has not been initialized` 예외가 발생한다. 

일반적으로 이 프로퍼티는 의존관계 주입(DI) 프레임워크와 함께 사용하는 경우가 많다.
