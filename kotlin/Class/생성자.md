코틀린에서는 주 생성자와 부 생성자를 구분한다. 또한 초기화 블록을 통해 초기화 로직을 추가할 수 있다. [함수](함수.md)와 마찬가지로 생성자 파라미터에는 기본값을 정의할 수 있다.

> 만약 모든 생성자 파라미터에 기본값을 지정하면 컴파일러가 자동으로 파라미터가 없는 생성자를 만든다. 만약 자바 코드에서 이러한 클래스의 생성자를 사용하려고 하면 기본값을 정의한 생성자에 [디폴트 파리미터 함수](함수.md#디폴트%20파리미터%20함수)처럼 `@JavmOverloads constructor`를 지정해야 한다.
## 주 생성자
```kotlin
class User(val nickname: String) // 주 생성자
class User constructor(val nickname: String)
class User(_nickname: String) {
    val nickname = _nickname
}
class User(val nickname, val isSubscribed: Boolean = true)
```
> `constructor` 키워드는 주 생성자나 부 생성자 정의를 시작할 때 사용한다.

클래스 이름 뒤에 오는 괄호로 둘러쌓인 코드를 주 생성자라고 한다. 주 생성자는 생성자 파라미터를 지정하고 그 생성자 파라미터에 의해 초기화되는 프로퍼티를 정의하는 2가지 목적에 쓰인다. 

또한, `constructor` 키워드를 통해 명시적으로 정의할 수도 있다. 하지만, 주 생성자 앞에 별도의 어노테이션이나 가시성 변경자가 없다면 `constructor`를 생략하는 것이 일반적이다. 
## 초기화 블록
```kotlin
class User constructor(_nickname) {
    val nickname: String
    
    init {
        nickname = _nickname
    }
}
```
`init` 키워드를 통해 초기화 블록을 사용할 수 있다. 초기화 블록에는 클래스의 객체가 만들어질 때 실행될 초기화 코드가 들어간다. 초기화 블록은 주 생성자와 함께 사용된다. 이는 주 생성자가 제한적이기 때문에 별도의 코드를 포함할 수 없기 때문이다.
## 부 생성자
```kotlin
open class Downloader {
    constructor(url: String?) { /* ... */}
    constructor(url: URL?) { /* ... */ }
}
```

일반적으로 코틀린에서는 생성자가 여럿 있는 경우가 자바에 비해서 매우 적다. 대부분의 경우 디폴트 파라미터로 모두 해결이 가능하기 때문이다. 하지만 필요한 상황이 있는 경우가 있어 부 생성자 기능을 제공한다. 단순히 클래스 선언 내부에 `constructor` 키워드를 이용해 정의하면 된다.

```kotlin
class MyDownloader : Downloader {
    constructor(url: String?) : super(url) { /* ... */ }
    constructor(url: URL?) : super(url) { /* ... */ }
}

class MyDownloader : Downloader {
    constructor(url: String?) : this(URL(url))
    constructor(url: URL?) : super(url) { /* ... */ }
}
```

`super` 키워드를 통해 상위 클래스의 생성자를 호출할 수 있다. 또한, `this`를 활용하여 자신의 다른 생성자를 호출할 수 있다.

만약, **클래스의 주 생성자가 없다면 모든 부 생성자는 반드시 상위 클래스를 초기화하거나 다른 생성자에게 위임해야 한다.**
## 생성자 호출
```kotlin
val alice = User("Alice")
val bob = user("Bob", false)
```

생성자를 호출할 때는 `new`와 같은 추가 키워드 없이 직접 호출하면 된다. 생성자 자체가 함수이기 때문이다. 

```kotlin
open class User(val nickname)
class SocialUser(nickname) : User(nickname)

open class Button
class RadioButton: Button()
```

또한, 기반 클래스의 생성자가 인자를 받아야 한다면 클래스의 주 생성자에서 기반 생성자를 호출해야 하며, 이는 인자가 없더라도 동일하다.
