상속되는 상위 클래스에 `sealed` 변경자를 붙이면 **봉인된 클래스**라고 하며, 그 상위 클래스를 상속한 하위 클래스의 가능성을 제한할 수 있다. `sealed` 클래스가 선언된 클래스에는 다음과 같은 조건을 따라야 한다.
- 직접적인 하위 클래스들은 반드시 컴파일 시점에 알려져야 한다.
- 직접적인 하위 클래스들은 상위 클래스가 선언된 패키지와 같은 패키지에 속해야 한다.
- 모든 하위 클래스가 같은 모듈 안에 위치해야 한다.

```kotlin
sealed class Expr
class Num(val value: Int) : Expr()
class Sum(val left: Expr, val right: Expr) : Expr()
```

봉인된 클래스를 사용하면 `when` 같은 분기에서 모든 하위 클래스를 처리한다면 디폴트 분기가 따로 필요 없다.

```kotlin
fun eval(e: Expr) : Int = 
    when (e) {
        is Num -> e.value
        is SUm -> eval(e.right) + eval(e.left)
    }
```

`sealed` 키워드는 기본적으로 추상 클래스임을 명시함으로 [`abstract`](접근%20변경자.md)를 붙일 필요가 없다. 그리고, 클래스가 아니라 [인터페이스](인터페이스.md) 앞에도 `sealed` 변경자를 붙여 봉인된 인터페이스를 만들 수 있다. 봉인된 인터페이스도 동일한 규칙을 따른다.
