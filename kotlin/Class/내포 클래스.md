클래스 안에서 다른 클래스를 선언한 경우 그 클래스를 내포 클래스라고 한다. 코틀린에서는 자바와 달리 내포 클래스는 명시적으로 요청하지 않는 한 바깥쪽 클래스 인스턴스에 대한 접근 권한이 없다. 

```kotlin
interface State: Serializable

interface View {
    fun getCurrentState: State
    fun restoreState(state: State) { /* ... */ }
}

class Button : View {
    override fun getCurrentState(): State = ButtonState()
    override fun restoreState(state: State) { /*...*/ }
    class ButtonState : State { /* ... */ }
}
```

> 위 코드를 자바에서 실행하면 `java.io.NotSerializableException: Button`이라는 오류가 발생하고, 이는 자바에서 `ButtonState` 클래스가 바깥쪽 `Button`에 대한 참조를 암시적으로 포함하기 때문이다. 이를 해결할려면 자바에서는 `ButtonState`를 `static` 클래스로 선언해야 한다.

코틀리에서 **내포 클래스는 아무런 변경자도 없으면 자바 `static` 내포 클래스와 같다.** 만약 바깥쪽 참조를 포함하게 하려면 `inner` 변경자를 붙여야 한다. 내포 클래스와 내부 클래스의 차이는 다음과 같다.

| 클래스 B 안에 정의된 클래스 A  | 자바               | 코틀린             |
| ------------------- | ---------------- | --------------- |
| 내포 클래스(바깥쪽 참조 저장 x) | `static class A` | `class A`       |
| 내부 클래스(바깥쪽 참조 저장)   | `class A`        | `inner class A` |

만약 내부 클래스에서 바깥쪽 클래스의 인스턴스를 참조하려면 내부 클래스 안에서 `this@클래스이름`으로 접근해야 한다.

```kotlin
class Outer {
    inner class Inner {
        fun getOuterReference(): Outer = this@Outer
    }
}
```
