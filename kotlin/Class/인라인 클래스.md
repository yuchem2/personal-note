```kotlin
@JvmInline
value class UsdCent(val amount: Int)
```

인라인 클래스는 성능을 희생하지 않고 타입 안전성을 얻기 위한 방법이다. `value` 키워드를 사용하고 `@JvmInline` 어노테이션을 붙여서 생성할 수 있다. 실행 시점에 인라인 클래스 인스턴스는 감싸진 프로퍼티로 대체된다. 즉, 클래스의 데이터가 사용되는 시점에 인라인 된다.

> 정확하게는 코틀린 컴파일러는 가능한 경우 최대한 항상 인라인 클래스를 내부 프로퍼티의 타입으로 표현한다. 래퍼 타입을 꼭 유지해야 하는 경우 중 가장 대표적인 예시는 인라인 클래스를 타입 파라미터로 사용하는 경우다. [문서](https://kotlinlang.org/doce/inline-classes.html)에서 자세하게 확인할 수 있다.

인라인 클래스를 선언하려면 다음과 같은 조건을 따라야 한다.
- 클래스가 프로퍼티를 하나만 가져야 하고, 주 생성자에서 초기화돼어야 한다.
- 인라인 클래스는 상속할 수도 없고, 다른 클래스를 상속할 수도 없다.

하지만, 인터페이스를 상속하거나 메서드를 정의하거나, 계산된 프로퍼티를 제공할 수 있다.
