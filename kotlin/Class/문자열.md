코틀린 문자열은 자바 문자열과 똑같다. 하지만 추가적인 [확장 함수](클래스%20확장.md)를 제공함으로써 표준 자바 문자열을 더 쉽게 다룰 수 있게 해준다.
## 문자열 나누기
`String`에는 대표적으로 `split` 함수가 있다. 하지만 자바 `split` 메서드로는 점`.`을 사용해 문자열을 분리할 수 없다. 왜냐하면 `split`은 정규식을 구분 문장열로 받아 그 정규식에 따라 문자열을 나누기 때문이다.

코틀린에서는 자바의 `split` 대신에 여러 가지 다른 조합의 파라미터를 받는 `split` 확장 함수를 제공함으로써 혼동을 야기하는 메서드를 감춘다. 정규식을 파라미터로 받는 함수는 `String`이 아니라 `Regex` 타입의 값을 받는다.

이를 통해 타입에 따라 정규식/일반 텍스트 중 어느 것으로 문자열을 분리하는 것인지 쉽게 알 수 있다.
## 정규식과 삼중 따음표

```kotlin
fun parsePath(path: String) {
    val directory = path.substringBeforeLast("/")
    val fullName = path.substringAfterLast("/")
    val fileName = fullName.substringBeforeLast(".")
    val extension = fullName.substringAfterLast(".")
    println("Dir: $directory, name: $fileName, ext: $extension")
}

fun parsePathRegex(path: String) {
    val regex = """(.+)/(.+)\.(.+)""".toRegex()
    val matchResult = regex.matchEntire(path)
    if (matchResult != null) {
        val (directory, filename, extension) = matchResult.destructured
        println("Dir: $directory, name: $fileName, ext: $extension")
    }
}
```
> - `String.substringBeforeLast()`: 어떤 문자열에서 구분 문자열이 맨 나중에 나타난 곳 전의 부분 문자열을 리턴
> - `String.substringAfterLas()`: 어떤 문자열에서 구분 문자열이 맨 나중에 나타난 곳 뒤의 문자열을 리턴
> - `Regex.matchEntire()`: 정규식에 일치하는 패턴 그룹의 모든 문자열을 찾음 

코틀린 정규식 문법은 자바와 동일하다. `toRegex` 확장 함수를 통해 문자열을 정규식으로 변환할 수 있다. 

일반 문자열을 사용하는 경우 특정 기호를 사용하려면 이스케이프해야하지만, 3중 따옴표 문자열(`"""~~""""`)을 사용하면 어떤 문자도 이스케이프하지 않아도 되기 때문에 유용하다.

```kotlin
val kotlinLogo = 
    """
    | //
    | //
    |/ \
    """.trimInput()
```
> `trimgInput()`:  문자열의 모든 줄에서 가장 짧은 공통 들여쓰기를 찾아 각 줄의 첫 부분에서 제거하고 공백만으로 이뤄진 첫번째 줄과 마지막 줄을 제거한다. 이때 코틀린은 CRLF, CR, LF를 모두 줄 끝으로 취급한다.

3중 문자열은 문자열 이스케이프뿐만 아니라 줄 바꿈을 포함한 아무 문자열이나 그대로 들어간다. 따라서 3중 따음표를 쓰면 여러 텍스트를 쉽게 코드에 포함시킬 수 있다.

```kotlin
val think = """Hmm ${"\uD83E\uDD14"}"""
```

또한, [문자열 템플릿](문자열%20템플릿.md) 또한 사용할 수 있다. 그러나 이스케이프할 수 없기 때문에 `$`나 유니코드 이스케이프를 사용하고 싶을 때에는 **내포 식**을 이용해야 한다.
