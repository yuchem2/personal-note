[Kotlin](Kotlin.md)에서 `interface` 키워드를 통해 인터페이스를 정의할 수 있다.

```Kotlin
interface Clickable {
    fun click()
}
```

코틀린 인터페이스는 추상 메서드뿐 아니라 구현이 있는 메서드도 정의할 수 있다. 하지만, 아무 상태도 들어갈 수 없다. 이 인터페이스를 상속하는 구현하는 모든 비추상 클래스는 `click`에 대한 구현을 제공해야 한다.

```kotlin
class Button : Clickable {
    override fun click() = println("I was clicked")
}
```

코틀린에서는 **상속**이나 **구성**에서 모두 클래스 이름 뒤에 콜론을 붙이고 인터페이스나 클래스 이름을 적는 방식을 사용한다. 하지만 인터페이스는 개수 제한이 없지만, 클래스는 오직 하나만 확장할 수 있다.

```kotlin
interface Focusable {
    fun setFocus(b: Boolean) = println("I ${if (b) "got" else "lost"} focus.")
    fun showOff() = println("I'm focusable!")
}
```

상위 클래스나 상위 인터페이스에 있는 [프로퍼티](프로퍼티.md)나 메서드를 오버라이드할 때는 `overrride`변경자를 반드시 사용해야 한다. 인터페이스 메서드는 디폴트 구현을 제공할 수 있다.

```kotlin
interface User {
    val nickname: String
}
```

코틀린에서는 인터페이스에 추상 프로퍼티 선언을 넣을 수 있다. 위와 같은 경우 `User` 인터페이스를 구현하는 [클래스](클래스.md)가 `nickname`의 값을 얻을 수 있는 방법을 제공해야 한다는 뜻이다. 

```kotlin
interface EmailUser {
    val email: String
    val nickname: String
        get() = email.substringBefore('@')
}
```

추상 프로퍼티뿐만 아니라 게터와 세터가 있는 프로터피를 선언할 수 있다. 하지만 뒷밤침하는 필드를 참조할 수는 없다. 위 인터페이스를 상속하는 하위 클래스에서는 `email`은 반드시 오버라이드해야 하지만, `nickname`은 상속받을 수 있다.
