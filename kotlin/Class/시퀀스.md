```kotlin
// map 결과 컬렉션이 생성된 후 filter 연산이 수행된다. 즉 리스트가 2개 생성된다.
people.map(Person::name).filter { it.startsWith("A") }
// 시퀀스를 활용하면 중간 리스트 없이 결과만 반환된다.
people
    .asSequence()
    .map(Person::name)
    .filter { it.startsWith("A") }
    .toList()
```

[Collections API](Collections%20API.md)를 연쇄하면 매 단계마다 중간 결과를 새로운 컬렉션에 임시로 담게된 된다. 이러한 임시 컬렉션은 다시 사용되지 않기 때문에 성능을 낭비할 수 있다. 이를 해결하기 위해 코틀린에는 시퀀스가 존재한다. 

자바 8의 스트림과 비슷하게 중간 임시 컬렉션을 사용하지 않고 컬렉션 연산을 연쇄하는 방법을 제공한다.

지연 계산 시퀀스는 `Sequence` 인터페이스부터 시작한다. 이 인터페이스는 `iterator`라는 하나의 메서드만 존재한다. 이 메서드를 활용하여 시퀀스에서 원소 값을 얻을 수 있다. `Sequence` 인터페이스의 특징은 다음과 같다.
- 시퀀스의 원소([Collections API](Collections%20API.md))는 필요할 때 게으르게 연산된다. (lazy evaluation)
- 어떤 컬렉션이든 시퀀스로 바꿀 수 있다.
- 정말 계산을 실행하게 만들려면, 시퀀스의 원소를 하나씩 이터레이션하거나 결과 시퀀스를 [Collections](Collections.md)으로 변환해야 한다.
## 연산 실행
```kotlin
// 최종 연산이 없으면 변환이 지연된다.
println(
    listOf(1, 2, 3, 4)
        .asSequence()
        .map { 
            print("map($it) ")
            it * it
        }.filter {
            print("filter($it) ")
            it % 2 == 0
        }
) // Kotlin.sequences.FilteringSequence@506e1b77
// 최종 연산이 있으면 모든 연산이 수행된다.
println(
    listOf(1, 2, 3, 4)
        .asSequence()
        .map { 
            print("map($it) ")
            it * it
        }.filter {
            print("filter($it) ")
            it % 2 == 0
        }.toList()
) 
// map(1) filter(1) map(2) filter(4) map(3) filter(9) map(4) filter(16) [4, 16]
```

시퀀스에 대한 연산은 중간 연산과 최종 연산으로 나뉜다. 
- 중간 연산: 다른 시퀀스를 반환. 이 시퀀스는 최종 시퀀스의 원소를 변환하는 방법을 안다. 항상 지연 계산된다.
- 최종 연산: 결과를 반환한다. 결과는 시퀀스에서 일련의 계산을 수행해 얻을 수 있는 컬렉션이나 원소, 수, 또 다른 객체다.

```kotlin
// 다음 연산은 1, 2 원소에 대해서만 수행되고 종료된다.
println(
    listOf(1, 2, 3, 4)
        .asSequence()
        .map { it * it }
        .find { it > 3 }
) // 4

// map 먼저 수행되고 filter가 수행되서 모든 people의 원소가 filter를 수행
println(
    people
        .asSequence()
        .map(Person::name)
        .filter { it.length < 4 }
        .toList()
)

// filter 먼저 수행되어 위의 연산보다 연산 횟수가 줄어듦
println(
    people
        .asSequence()
        .filter { it.length < 4 }
        .map(Person::name)
        .toList()
)
```

시퀀스의 경우 **모든 연산은 각 원소에 대해 순차적으로 적용된다.** 즉, 첫 번째 원소가 처리되고 두 번째 원소가 처리된다. 그러므로 시퀀스를 활용하게 되면 지연 계산으로 인해 **원소 중 일부의 계산은 이뤄지지 않는다.** 이로 인해 연쇄적인 연산에서 더 빨리 원소들을 제거하면 할수록(코드 로직이 달라지지 않는 한) 코드의 성능이 더 좋아진다.
## 시퀀스 생성
```kotlin
val naturalNumbers = generateSequence(0) { it + 1 }
val numberTo100 = naturalNumbers.takeWhile { it <= 100 }
println(numbersTo100.sum()) // 5050 <- 여기서 연산이 수행된다.
```
시퀀스는 모든 컬렉션에 대해 `asSequence` 함수가 있어 이를 통해 만들 수 있다. 하지만, `generateSequence` 함수를 사용하여 시퀀스를 생성할 수 있다.
